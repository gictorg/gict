2. DATABASE STRUCTURE
================================================================================

CORE TABLES:
------------

1. INSTITUTES TABLE
   - id (Primary Key)
   - name (Institute name)
   - slug (URL-friendly identifier)
   - address, phone, email, website
   - logo_url, banner_url
   - status (active/inactive/pending)
   - created_at, updated_at

2. USERS TABLE (Enhanced)
   - id (Primary Key)
   - institute_id (Foreign Key to institutes)
   - username, email, password
   - full_name, phone, address
   - profile_image
   - user_type (super_admin/admin/student/faculty)
   - status (active/inactive)
   - created_at, updated_at

3. COURSES TABLE (Enhanced)
   - id (Primary Key)
   - institute_id (Foreign Key to institutes)
   - name, description, category
   - duration_months, fee
   - status (active/inactive)
   - created_at, updated_at

4. SUB_COURSES TABLE (New)
   - id (Primary Key)
   - course_id (Foreign Key to courses)
   - name, description
   - duration_weeks, fee
   - status (active/inactive)
   - created_at, updated_at

5. STUDENT_ENROLLMENTS TABLE (Enhanced)
   - id (Primary Key)
   - user_id (Foreign Key to users)
   - sub_course_id (Foreign Key to sub_courses)
   - enrollment_date, completion_date
   - status (active/completed/dropped)
   - final_marks, certificate_url
   - created_at, updated_at

6. PAYMENTS TABLE (Enhanced)
   - id (Primary Key)
   - user_id (Foreign Key to users)
   - sub_course_id (Foreign Key to sub_courses)
   - amount, payment_date
   - payment_method, transaction_id
   - status (pending/completed/failed)
   - receipt_url
   - created_at, updated_at

7. STUDENT_DOCUMENTS TABLE
   - id (Primary Key)
   - user_id (Foreign Key to users)
   - document_type, document_name
   - imgbb_url, file_extension
   - status (pending/approved/rejected)
   - uploaded_at

RELATIONSHIPS:
--------------
- Institutes â†’ Users (One-to-Many)
- Institutes â†’ Courses (One-to-Many)
- Courses â†’ Sub_Courses (One-to-Many)
- Users â†’ Student_Enrollments (One-to-Many)

- Users â†’ Payments (One-to-Many)

5. INSTALLATION & SETUP
================================================================================

STEP 1: Database Setup
---------------------
1. Create a MySQL database
2. Import the database structure:
   - Option A: Run database_franchise_structure.sql
   - Option B: Run migrate_to_franchise.php (for existing data)

STEP 2: Configuration
---------------------
1. Update config/database.php with your database credentials
2. Ensure all required PHP extensions are enabled
3. Set up ImgBB API key for image uploads

STEP 3: File Permissions
-----------------------
1. Ensure upload directories are writable
2. Set proper file permissions for security

STEP 4: Migration (If Upgrading)
--------------------------------
1. Backup existing database
2. Run migrate_to_franchise.php
3. Verify data migration
4. Test all functionality

STEP 5: Testing
---------------
1. Test login with demo credentials
2. Verify all dashboards work correctly
3. Test institute-specific functionality
4. Verify data isolation

   - Ensure all required files are present

3. Image Upload Issues:
